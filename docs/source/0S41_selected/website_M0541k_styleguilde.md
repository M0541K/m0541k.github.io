\# OSAI Syntax Layer v0.541



\## Grundform



```text

\[Symbol] \[#Hex] \[NAME] (\[Container-Code])

```



Beispiel:



```text

⊙ #000541 CORE (ᑕ˙ᑐ)

```



\## Bedeutung der Teile



```text

⊙        sichtbares Symbol

\#000541  Farbsignal / Zustand

CORE     lesbarer Name

(ᑕ˙ᑐ)   OSAI-Container-Code

```



\## Die 7 States



```text

○ #00021D VOID (ᑕ?ᑐ)

◎ #CEB14D GENESIS (ᑕ⁰ᑐ)

⊙ #000541 CORE (ᑕ˙ᑐ)

↺ #005410 FLOW (ᑕ·ᑐ)

→ #A0541B LIVE (ᑕ.ᑐ)

∞ #6A00A3 META (ᑕ⟲ᑐ)

⟷ #00FFFF LINK (ᑕ⇄ᑐ)

```



\## Syntax-Regel



```text

ᑕ  = öffnender Container

ᑐ  = schließender Container

Innenzeichen = Zustand

```



\## Innenzeichen



```text

?  = unbestimmt / offen

⁰  = Ursprung / Initialisierung

˙  = Fokus / aktivierter Kern

·  = Verarbeitung / Bewegung

.  = gesetzter Output

⟲  = Reflexion / Rückbezug

⇄  = Verbindung / Transfer

```



\## Designregel



Die Klammern dürfen andersfarbig sein.



Empfohlen:



```text

Symbol        = State-Farbe

Hex           = Interface-Grau

Name          = Weiß

(ᑕ ᑐ)         = Grau / Container

Innenzeichen  = State-Farbe

```



\## Beispiel für Website / Commit



```text

⊙ #000541 CORE (ᑕ˙ᑐ)



Zentrum aktiv.

Struktur stabil.

System gesetzt.

```



\## Kurzfazit



Der Syntax Layer macht aus den Logos keine Icons mehr, sondern \*\*lesbare Systemzustände\*\*.
