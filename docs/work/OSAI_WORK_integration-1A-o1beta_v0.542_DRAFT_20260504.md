OSAI 1A.o1beta ist der erste bewusste Integrationsraum für alle Elemente,
die im OSAI/M0541K-System bereits existieren,
aber noch nicht endgültig verlinkt, eingeordnet oder veröffentlicht sind.

Diese Datei ist keine neue CORE-Datei.

## 2. Grundregel

CORE bleibt stabil.
WORK bewegt sich.
RELEASE wird veröffentlicht.
ARC erinnert.

OSAI 1A.o1beta dient als Brücke zwischen:

- frühen Werkspuren
- Bildquellen

- Zeichen
- Archivnotizen
- Website-Strukturen
- offenen Begriffen
- späteren Release-Kandidaten


## 3. Was hier hinein darf

In OSAI 1A.o1beta dürfen Elemente aufgenommen werden, wenn sie:

- sichtbar oder belegbar sind
- für M0541K / OSAI eine Bedeutung haben könnten
- noch nicht endgültig im CORE sitzen
- für ARTCORE, ARC, WEB, STYLEGUIDE oder RELEASE relevant werden können
- als Ursprung, Signal, Fragment oder offene Spur wichtig sind


## 4. Was hier nicht passieren soll

OSAI 1A.o1beta soll nicht:

- den CORE überschreiben
- ungeprüfte Begriffe endgültig versiegeln
- jedes Bild automatisch öffentlich machen
- private Bildpool-Elemente unkontrolliert veröffentlichen
- aus jedem Fundstück sofort eine Theorie machen

## 6. Entscheidungslogik

Jedes neue Element bekommt zunächst einen Status:

OPEN
= gefunden, aber noch nicht geprüft

DRAFT
= beschrieben, aber noch nicht eingeordnet

LINKED
= sinnvoll mit einem Bereich verbunden

ARC
= als Zeitspur dokumentiert

ARTCORE
= als ausgewähltes Werk relevant

CORE-CANDIDATE
= könnte später in den Kern, ist aber noch nicht versiegelt

RELEASE-CANDIDATE
= könnte später veröffentlicht werden

SEALED
= bewusst versiegelt


## 7. Prüfregel

Ein Element wandert erst weiter, wenn klar ist:

1. Was ist es?
2. Woher kommt es?
3. Ist es belegbar?
4. Ist es privat oder öffentlich?
5. Gehört es zu CORE, ARTCORE, ARC, WEB, STYLEGUIDE oder RELEASE?
6. Muss es erklärt werden oder darf es offen bleiben?
